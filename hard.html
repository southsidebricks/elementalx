Main Components

Identify parts and their use

The motherboard is the basis of what is a computer. It is the main circuit board and holds the CPU, inputs, and outputs.

The CPU, or, Central Processing Unit, is the brains of the computer. All data is processed through this small chip, found on the motherboard.

The Hard Drive is the main storage device for computers. There are HDD's and SSD's, which are the two main types of hard drives.

The Power Supply is what powers the computer. When troubleshooting PC's, an unplugged cable is often the problem.

RAM, or Random Access Memory, is an example of an expansion card and computer data storage. It is often faster than Hard Drives for accessing memory.

Storage

RAM, Hard Drives, RAID

Understanding how storage devices vary in use will allow you to choose the right one to fit your needs.

  • Expansion Cards: any circuit board added to the motherboard for additional features (Video Card, RAM, Sound Card)

  • RAM: volatile memory, meaning that when the computer is shut down, all memory is erased.

  • ROM: non-volatile memory. Read Only Memory cannot be modofied or overwritten, so a computers ROM has the essential and undeletable instructions for booting the PC.

  • HDD: Hard Disk Drive is non-volatile. It uses rotating disks to write memory with magnetic heads, and is slower than SSD's.

  • SSD: Solid State Drives have no moving mechanical components, so they are just external blocks of storage, usually faster than an HDD.

  • RAID: RAID 5 stripes data across three or more drives and uses parity checking, so that if one drive fails, the other drives can re-create the data stored on the failed drive by using the parity information.

Troubleshooting

Assess, diagnose, and find solutions

The basic theory of troubleshooting uses a method to assess, diagnose, and find a solution to any IT related issues. Taking a step-by-step approach will ensure that no details are missed and the solution can be found quickly and accurately.

  1. Identify the Problem
  2. Establish a theory of probable cause
  3. Test the theory to determine the cause
  1. Establish a plan of action
  2. Implement the plan
  3. Verify full system functionality
  4. Document findings
